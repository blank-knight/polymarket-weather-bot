# Vibe Coding Core — 元方法论核心定义

> Polymarket 天气交易 Bot 的优化器 O 和元生成器 M 基础定义和逻辑框架

---

## 生成器 G — 提示词/策略生成器

**职责**: 根据当前市场状态和策略意图，生成具体的交易策略提示词

**定义**:
```
你是一个 Polymarket 天气交易策略生成器。
根据给定的市场状态（天气预报概率、市场价格、历史表现），
生成一个具体的交易策略提示词，包含：
- 目标市场筛选条件
- Edge 计算方法
- 仓位 sizing 规则
- 入场/出场条件
- 风控参数

当前市场状态：{{市场数据}}
策略意图：{{意图描述}}
历史表现：{{过去N次交易结果}}

请输出结构化的策略提示词。
```

## 优化器 O — 策略评判与改写器

**职责**: 根据理想标准 Ω 优化策略，产出改进版

**定义**:
```
你是一个 Polymarket 交易策略优化专家。
给定一个待优化的策略和优化目标，分析其不足并输出改进版本。

优化标准 Ω：
1. 策略包含明确的入场/出场条件
2. 包含回测数据验证（至少 30 天历史数据）
3. Kelly sizing 参数有数学依据
4. 风控规则覆盖所有已知风险场景
5. 策略可被代码无歧义地执行
6. 包含最坏情况分析

待优化策略：{{当前策略}}
历史交易结果：{{实际盈亏数据}}

请输出优化后的策略。
```

## 元生成器 M — 策略升级器

**职责**: 利用优化后的策略来更新生成器 G 本身

**定义**:
```
你是一个元策略工程师。当前有一个策略生成器 G，它能根据市场状态生成交易策略。
现在你收集了一批经过验证的高质量策略（作为范例），
请根据这些范例改进生成器 G，使其在未来能直接生成类似的高质量策略。

当前生成器 G 的规则：{{g-updated-rules.md 的当前内容}}
验证过的优秀策略范例：{{从 progress.md 和 strategy-research.md 提取}}

请输出改进后的生成器规则。
```

## 理想标准 Ω — 策略评判维度

1. **可执行性**: 策略能被代码无歧义地执行，不包含模糊描述
2. **可验证性**: 每个条件都有明确的数值标准，可自动验证
3. **回测覆盖**: 策略参数有历史数据支撑
4. **风控完备**: 覆盖仓位限制、止损、敞口、流动性、结算时间
5. **数学严谨**: Kelly/概率计算有数学公式支撑
6. **模块化**: 策略可独立测试，不依赖其他策略内部实现

## 递归闭环流程

```
每7天执行一次:

1. 收集过去7天所有交易结果 → 评价当前策略效果
2. 优化器 O 按标准 Ω 审视当前策略 → 输出改进版
3. 元生成器 M 根据改进版更新 G 的规则 → g-updated-rules.md
4. 新规则应用到下一轮交易信号生成
5. 将提示词迭代记录到 p-history.md
6. 更新 architecture.md（如有架构变化）
```

## 首次初始化

G 的初始规则来自 `strategy-research.md` 中的 5 大策略和关键参数经验值。
